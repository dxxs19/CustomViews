常用设计模式：

应用最广的模式——singleton(单例模式) ['sɪŋg(ə)lt(ə)n]
自由扩展你的项目——builder(建造者模式)
使程序运行更高效——prototype(原型模式) ['prəʊtətaɪp]
解决解耦的钥匙——observer(观察者模式) [əb'zɜːvə]
应用最广泛的模式——factory(工厂模式)
创建型设计模式——abstractfactory(抽象工厂模式)
时势造英雄——strategy(策略模式) [ˈstrætədʒɪ]
统一编程接口——facade(外观模式)  [fə'sɑːd]


走向灵活软件之路——面向对象的六大原则：
                    1.优化代码的第一步——单一职责原则；
                    2.让程序更稳定、更灵活——开闭原则；
                    3.构建扩展性更好的系统——里氏替换原则；
                    4.让项目拥有变化的能力——依赖倒置原则；
                    5.让系统有更高的灵活性——接口隔离原则；
                    6.更好的可扩展性——迪米特原则。
